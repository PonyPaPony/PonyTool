## [2.0.0] — 2025-12-26

### Added
- Новая команда `pony bootstrap` для создания базовых файлов проекта:
  - `.gitignore`
  - `README.md`
  - `LICENSE` (MIT, с текущим годом)
- Поддержка генерации структуры проекта через `.ponyinit`
- Поддержка запуска тестов с coverage и HTML-отчётом:
  - `pony test --coverage`
  - `pony test --coverage --html`

### Changed
- Полностью переписана команда `pony init`
  - упрощён контракт
  - убрана магия и интерактив
  - повторный запуск стал безопасным и воспроизводимым
- Полностью переписана команда `pony deps`
  - зависимости определяются по реальным импортам проекта
  - стандартная библиотека исключается автоматически
  - `pip freeze` больше не используется
- CLI стал плоским и предсказуемым:
  - `pony init`
  - `pony deps`
  - `pony bootstrap`
  - `pony test`

### Deleted
- Интерактивные меню и подтверждения
- Неявные действия и автодогадки
- Лишние абстракции в CLI

### Примечания
- Версия 2.0.0 является архитектурным обновлением проекта.
- PonyTool остаётся небольшим CLI-инструментом и не стремится быть фреймворком.

## [1.1.1] - 2025-03-10
### Added
- `pony project status`:
  - проверка Python и virtualenv
  - состояние Git-репозитория
  - наличие ключевых файлов проекта
  - краткий (`--short`) и JSON (`--json`) режимы
- `pony test doctor`:
  - проверка наличия pytest
  - поиск директории tests/
  - обнаружение test_*.py файлов
  - проверка pytest cache
- сохранение результата последнего запуска тестов
- использование отчёта тестов в `project status`

### Changed
- унификация CLI-вывода во всех командах
- отказ от жёсткой логики путей и окружений
- улучшена обработка случаев без upstream в Git
- согласование поведения doctor-команд

### Fixed
- исправлены падения при отсутствии remote или upstream
- устранены ложные срабатывания в git doctor
- корректная обработка пустых окружений
- исправлены ошибки CLI-аргументов и парсеров
